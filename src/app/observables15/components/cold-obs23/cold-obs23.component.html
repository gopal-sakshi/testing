Cold Observables

  • you want each subscription to receive all the data from the beginning
  • When you want to repeat the data for each new subscriber
  • Examples: HTTP requests, mouse clicks, timer observables
Angular's http returns Cold Observables
Any new subscriptions will trigger the execution once again
- meaning the request will be send as many times as there are subscriptions
- To avoid this use share,shareReplay, publish, publishLast or publishReplay
